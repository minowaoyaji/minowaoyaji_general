# GitHub Pages 公開設計

## 目的

リポジトリ内の `test` に記載された `test` と `test2` を、GitHub Pages のプロジェクトサイトとしてブラウザから閲覧できるようにする。

公開先は `https://minowaoyaji.github.io/minowaoyaji_general/` とする。

## 公開方式

リポジトリのルートに静的な `index.html` を追加し、`main` ブランチのルートを GitHub Pages の公開元に設定する。ビルド処理や GitHub Actions は追加しない。

この方式は、現在の単一ファイルに近い小さな構成で最も単純であり、今後の更新も `index.html` を変更して `main` に push するだけで反映できる。

## ファイル構成

- 既存の `test` は変更せず残す。
- 新しい `index.html` に、見出しなどを加えず `test` と `test2` を1行ずつ表示する。
- 日本語などを将来追加しても扱えるよう、HTML文書は UTF-8 とする。

## 公開手順

1. `index.html` を追加し、ローカルで内容を確認する。
2. 変更を `main` ブランチへコミットして GitHub に push する。
3. GitHub Pages の公開元を `main` ブランチの `/` に設定する。
4. 公開処理の完了後、公開URLへアクセスして HTTP 応答と表示内容を確認する。

## エラー時の扱い

- GitHub の認証が利用できない場合は、認証が必要であることを報告して停止する。
- Pages が既に別の公開元に設定されている場合は、既存設定を確認し、この設計と競合するときは上書き前に報告する。
- 公開処理が進行中の場合は、完了まで短時間待ってから再確認する。

## 完了条件

- `main` ブランチに `index.html` が存在する。
- GitHub Pages が `main` ブランチのルートから公開する設定になっている。
- 公開URLが正常に応答し、`test` と `test2` が表示される。
